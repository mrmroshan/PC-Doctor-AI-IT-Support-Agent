# Security Policy

## Supported Versions

This project is currently in active development. Security fixes are applied to
the latest version on the default branch.

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly:

1. Open a GitHub issue with minimal details and mark it as security-related.
2. Do not include exploit code, secrets, or sensitive system data.
3. Share reproduction steps privately if requested by the maintainer.

## Response Expectations

- Initial triage target: within 7 business days
- Confirmed issues will be prioritized by severity and user impact
- Fixes may include code changes, documentation updates, or usage warnings

## Scope

Please report issues related to:

- Command execution safety
- Prompt-injection or instruction-confusion risks
- Secret handling and credential exposure
- Privilege escalation paths
- Unsafe default behavior that can harm user systems
